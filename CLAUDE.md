## Common Commands

### Setup
- Install dependencies for the whole monorepo:
  ```
  pnpm install
  ```
- Copy the example environment file and fill in required values:
  ```
  cp .env.example .env
  ```

### Development Servers
- Start the API server (backend):
  ```
  pnpm --filter @edu/server dev
  ```
- Start the web frontend (Next.js):
  ```
  pnpm --filter @edu/web dev
  ```
- Start the mobile app (React Native/Expo – placeholder script):
  ```
  pnpm --filter @edu/mobile dev
  ```
  *(The mobile dev script currently prints a placeholder; replace with the actual Expo/React‑Native command as needed.)*

### Build
- Build all packages:
  ```
  pnpm build
  ```
- Build a specific app:
  ```
  pnpm --filter @edu/web build   # web (Next.js)
  pnpm --filter @edu/server build   # server (TS → JS)
  pnpm --filter @edu/mobile build   # mobile (placeholder)
  ```

### Lint & Typecheck
- Lint the whole repo:
  ```
  pnpm lint
  ```
- Type‑check the whole repo:
  ```
  pnpm typecheck
  ```

### Tests
- Run all tests (Turbo will invoke the test script of each package):
  ```
  pnpm test
  ```
- Run server tests only (uses Vitest):
  ```
  pnpm --filter @edu/server test
  ```
- Run a single test file in the server package (replace `<path>` with the test file, relative to `apps/server`):
  ```
  pnpm --filter @edu/server test -- <path>
  ```
  Example:
  ```
  pnpm --filter @edu/server test -- src/controllers/teacher.controller.test.ts
  ```
- Mobile and web packages currently have placeholder test scripts; once real tests are added they can be run similarly.

### Formatting & Cleaning
- Format the entire codebase with Prettier:
  ```
  pnpm format
  ```
- Verify formatting without changes:
  ```
  pnpm format:check
  ```
- Clean build artefacts and caches:
  ```
  pnpm clean
  ```

### Database & Prisma
- Run database migrations (Postgres required):
  ```
  pnpm --filter @edu/server prisma:migrate
  ```
- Open Prisma Studio UI:
  ```
  pnpm --filter @edu/server prisma:studio
  ```
- Seed the database with development data:
  ```
  pnpm --filter @edu/server db:seed
  ```

### JWT Utilities
- Mint a development JWT (uses `scripts/mint-jwt.ts`):
  ```
  pnpm --filter @edu/server jwt:mint
  ```

## High‑Level Architecture

- **Monorepo managed with `pnpm` workspaces** – all packages live under `apps/` and share dependencies via the root `pnpm-lock.yaml`.
- **Three main applications**:
  1. **Server (`@edu/server`)** – Fastify based API written in TypeScript. Uses Prisma for Postgres, Redis for caching, Cloudinary for media storage, and a suite of AI pipelines (Claude‑driven) for insights. Middleware includes CORS, rate limiting, security headers, logging, and role‑based access.
  2. **Web (`@edu/web`)** – Next.js (React) front‑end. Communicates with the server through the `NEXT_PUBLIC_API_URL` environment variable. Includes UI components, dashboard pages, admin and teacher sections, and AI‑driven insight visualisations.
  3. **Mobile (`@edu/mobile`)** – React Native/Expo application (placeholder scripts). Mirrors much of the web UI for on‑the‑go access, using the same API endpoints.
- **Shared tooling**: ESLint, Prettier, TypeScript, Turborepo for task caching and parallel execution.
- **CI/CD**: GitHub Actions workflows (`.github/workflows/*.yml`) handle CI (lint, test, typecheck), build, Docker image creation, and deployment for server, web, and mobile.
- **AI Insight Engine**: Under `apps/server/src/ai/` lives a collection of prompt‑based pipelines that generate student, class, and teacher insights. These are invoked by scheduled jobs (`apps/server/src/jobs/`) and exposed via API routes.
- **Event‑Driven Design**: A simple event bus (`events/`) decouples domain actions (e.g., attendance updates) from side‑effects such as notifications or AI jobs.

## Development Tips
- Use `turbo run <task> --filter <package>` to focus on a single app while still benefiting from Turbo's caching.
- When adding new dependencies, run `pnpm add -w <pkg>` at the root to add to the workspace (or `pnpm add <pkg> -F @edu/server` for a specific package).
- Follow the existing folder conventions: `src/routes/v1/` for API endpoints, `src/controllers/` for request handling, `src/services/` for business logic, `src/middleware/` for request‑level concerns.
- Environment variables are defined in `.env.example`. Ensure `JWT_SECRET` is a strong secret and never commit real secrets.
- Tests for the server use Vitest; you can run `pnpm --filter @edu/server test:watch` during development.
